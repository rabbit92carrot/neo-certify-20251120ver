# Pages Directory

This directory contains all page components organized by role.

## Structure

- **auth/**: Authentication pages (Login, Register, PasswordReset)
- **manufacturer/**: Manufacturer dashboard and pages
- **distributor/**: Distributor dashboard and pages
- **hospital/**: Hospital dashboard and pages
- **admin/**: Admin dashboard and pages
- **mock/**: Mock KakaoTalk notification page

## Guidelines

1. Each page should be a top-level route
2. Use role-based access control
3. Import components from ../components
4. Handle loading and error states
5. Follow consistent layout structure
