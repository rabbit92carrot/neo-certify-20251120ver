# Post-MVP 기능 (2차 개발)

## 📋 Overview

이 폴더는 **PRD에서 "2차 개발"로 분류된 기능들**의 개발 계획을 포함합니다. MVP (1차 개발) 완료 후 우선순위에 따라 순차적으로 구현될 기능들입니다.

---

## 🎯 2차 개발 범위 (PRD Section 7.2)

PRD 1.2의 Section 7.2에서 명시한 2차 개발 기능들:

| 기능 | PRD 명시 | 우선순위 |
|------|---------|---------|
| 카카오 알림톡 API 연동 | 실제 발송, 비즈니스 채널 개설 필요 | 높음 |
| SMS 대체 발송 | 알림톡 실패 시 | 높음 |
| 폐기 통계/리포트 | 폐기 사유 분석, 통계 (기본 폐기는 Phase 5.4 MVP) | 중간 |
| 회수 로그 작성 | 모든 이력에 대한 회수 로그 작성 (기본 회수는 Phase 5.3 MVP) | 중간 |
| 환자 이력 조회 페이지 | 알림톡 링크로 본인 확인 | 중간 |
| 통계/리포트 | 제품별, 조직별 통계 | 낮음 |
| 재고 알림 | 재고 부족 시 알림 | 낮음 |
| 사용기한 임박 알림 | Lot 사용기한 경고 | 낮음 |
| 사업자등록번호 API | 진위 확인 API 연동 | 낮음 |
| 개인정보 암호화 | 환자 전화번호 암호화 | 낮음 |
| 모바일 반응형 UI | 태블릿/모바일 대응 | 낮음 |
| Safari 지원 | 브라우저 확장 | 낮음 |
| 백업/복구 | 데이터 백업 정책 | 낮음 |
| 히스토리 수정 기능 | 이력 수정 가능 (권한 제한) | 낮음 |

## 🚀 구현 우선순위

### High Priority (MVP 직후 즉시)
1. **카카오 알림톡 API 연동**
   - 비즈니스 채널 개설
   - 실제 발송 로직 구현
   - Mock 페이지 대체

2. **SMS 대체 발송**
   - 알림톡 실패 시 SMS로 대체
   - SMS 서비스 (예: Aligo, Toast) 연동

### Medium Priority (실제 사용 후 필요성 검토)
3. **폐기 기능** (현재 Phase 5.4에 있음 - 이동 고려)
   - 병원에서 제품 폐기 처리
   - Virtual Code 상태: DISPOSED

4. **환자 이력 조회 페이지**
   - 본인 확인 (전화번호 + 인증코드)
   - 알림톡 링크에서 접근
   - 본인이 받은 제품 이력 조회

5. **회수 기능 확장**
   - 24시간 이후에도 회수 가능 (관리자 권한)
   - 대량 회수 시뮬레이션

### Low Priority (안정화 이후)
6. **통계 & 리포트**
   - 제품별 생산/사용/재고 통계
   - 조직별 활동 리포트
   - 대시보드 차트

7. **재고 알림**
   - 재고 임계값 설정
   - 재고 부족 시 이메일/알림톡 발송

8. **사용기한 경고**
   - Lot 사용기한 임박 시 알림
   - 만료 예정 재고 자동 알림

9. **백업 & 복구**
   - 정기 백업 스케줄
   - 복구 프로세스 문서화
   - Point-in-time recovery

10. **시스템 개선**
    - 모바일 반응형 UI
    - Safari 브라우저 지원
    - 사업자등록번호 API 연동
    - 환자 전화번호 암호화
    - 히스토리 수정 기능

## 📝 Notes

### MVP에서 제외된 이유
1. **비용**: 카카오/SMS API는 실제 비용 발생
2. **복잡도**: 일부 기능은 MVP 검증 후 필요성 판단
3. **시간**: MVP 빠른 출시를 위해 핵심 기능에 집중
4. **검증**: 실사용 후 개선점 파악하여 2차 개발 반영

### 구현 시 고려사항
- 기존 코드 최소 수정 (새 기능 추가)
- 하위 호환성 유지
- 단계적 롤아웃 (Feature Flag 활용)
- A/B 테스트 가능하도록 설계

## 🎯 Success Criteria

2차 개발 완료 시:
- [ ] 카카오 알림톡 실제 발송 성공률 95% 이상
- [ ] SMS 대체 발송 자동화
- [ ] 환자 이력 조회 페이지 접근 가능
- [ ] 통계 대시보드 제공
- [ ] 재고 알림 자동화
- [ ] 백업 자동화 (일 1회)
- [ ] 모바일 반응형 UI (768px 이하)
- [ ] Safari 브라우저 완벽 지원
